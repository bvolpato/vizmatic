Theme-aware visuals from structured scenes.

Vizmatic gives models a small, typed visual language: flows, cards, graphs, timelines, icons, matrices, charts, slide frames, and animated GIFs. Satori and resvg handle rendering. Your design system stays in control.

API

Write scenes, not SVG paths.

Use React primitives with semantic tones. Render to PNG in Node for docs, websites, or generated decks. Scene titles and subtitles are optional when the visual should stand alone.

width = 1040;
height = 560;

<Scene title="Agent pipeline">
  <Flow stages={[
      { title: "Prompt", tone: "blue" },
      { title: "Scene spec", tone: "purple" },
      { title: "PNG / GIF", tone: "green" },
    ]} />
</Scene>
Backgrounds

Alpha by default, theme fill when requested.

PNG and SVG renders keep the canvas transparent unless a frame sets background={c.bg} or the render command asks for one. Omitted CLI dimensions start at 960x540 and grow to fit content on overflow; generated default-size wrappers do too. Add autoSize = false only when strict clipping errors are desired. Titleless scenes are valid.

# alpha-transparent PNG
vizmatic ./badge.tsx --out ./dist/frames --theme dark,light

# opaque theme background
vizmatic ./frame.tsx --out ./dist/frames --theme dark,light --background theme
Animated GIF

Export motion from the same scene language.

Static frames use create(theme). Animated frames export createScenes(theme) and render with vizmatic gif. Perfect for showing state machines, learning steps, product flows, and release timelines.

export function createScenes(theme) {
  return [
    { element: step(theme, 0), duration: 760, transition: "appear" },
    { element: step(theme, 1), duration: 760, transition: "fade" },
    { element: step(theme, 2), duration: 1100, transition: "fade" },
  ]
}

vizmatic gif frames/animated.tsx --out public/vizmatic --theme dark,light --watermark Acme --watermark-image ./logo.svg

Quality

Generated assets are treated like product code.

Vizmatic ships with automation that renders every example, validates docs, checks package contents, and gives maintainers a clean release path.

CI

Full verification on every push

./test.sh runs install, lint, Vitest, gallery rendering, docs checks, npm audit, package dry-run, and generated-file drift before a change is considered green.

Pages

Website generated from examples

The gallery, source viewer, dark/light assets, GIFs, and prompt preview are rebuilt from repo sources before GitHub Pages deploys.

Release

npm publishing with provenance

A manual release workflow bumps the version, runs the full gate, publishes to npm, pushes the tag, and creates a GitHub release.

Maintenance

Dependency and PR hygiene

Dependabot watches npm and GitHub Actions. pnpm holds timestamped package releases for seven days before selecting them.

Themes

Semantic tones, portable brand adapters.

Primitives consume tokens like positive, critical, purple, and cyan. Swap palettes without rewriting diagrams. Render dark and light variants from the same frame module.

  • Shared typography and spacing
  • Wrapping-safe labels and cards
  • Overflow detection before files are written
  • Optional watermark text, image/icon, and position per render
